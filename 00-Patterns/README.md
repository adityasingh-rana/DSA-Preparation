# 🧠 DSA Patterns Guide

> This folder contains core problem-solving patterns used across multiple DSA problems.
> The goal is to recognize patterns quickly and apply them efficiently during problem solving.

---

## 🎯 Purpose

- Identify patterns in problems quickly
- Build reusable logic templates
- Improve problem-solving speed
- Strengthen interview thinking

---

## 📌 How to Use This

For every problem:
1. Identify the pattern
2. Solve using the pattern
3. Revisit pattern notes
4. Update if new learning

> Pattern recognition > memorizing solutions

---

## 🔥 Patterns Covered

### 1. Two Pointers
- Used in sorted arrays
- Helps reduce time complexity
- Common in pair/triplet problems

📄 See: `two-pointers.md`

---

### 2. Prefix Sum
- Used for range queries
- Helps avoid repeated calculations
- Useful in subarray problems

📄 See: `prefix-sum.md`

---

### 3. Hashing
- Used for fast lookup
- Frequency counting
- Complement-based problems (like Two Sum)

📄 See: `hashing.md`

---

### 4. Kadane’s Algorithm
- Used for maximum subarray problems
- Optimizes brute force from O(n²) to O(n)

📄 See: `kadane.md`

---

## 🧠 Pattern Recognition Strategy

Ask yourself:
- Can I use two pointers?
- Is this a range query → prefix sum?
- Do I need fast lookup → hashing?
- Is this optimization → Kadane?

---

## ⚡ Golden Rule

> If you cannot identify the pattern,
> you have not understood the problem yet.
